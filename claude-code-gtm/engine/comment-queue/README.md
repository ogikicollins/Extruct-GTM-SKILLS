# Comment Queue — SELLL.io
> Managed by: `linkedin-automation` skill | Approval required: daily (5 minutes)
> Purpose: T-1 pre-engagement LinkedIn comments, generated by Claude and queued for Aaron's approval before Expandi posts them.

T-3 (follow) and T-2 (like) are fully automated by Expandi. T-1 comments require a human approval step because a comment is a public statement — it must read like Aaron wrote it personally.

**Daily approval window:** Review and approve/reject by 6 PM local time. If not approved by 6 PM, the comment is automatically skipped. Expandi proceeds to Email 1 on Day 0 without the T-1 comment. Never delay a campaign send for a missing comment.

---

## Queue Files

One file per date: `[YYYY-MM-DD].md`

Each file contains all comments queued for that day's pre-engagement window.

**File format:** `2026-06-21.md`

---

## Comment Format (per contact)

```markdown
### [Company] — [Contact Name] ([Title])
**Post:** [URL or brief description of post they made]
**Proposed comment:**
> [The comment text — 1-2 sentences, sounds like Aaron, references their specific post, adds a thought or question that invites reply]

**Approve?** [ ] Yes  [ ] Skip
```

---

## Comment Writing Rules (Claude generates these)

- Maximum 2 sentences
- Never generic ("great post!", "love this")
- Must reference something specific from the post
- Should add a thought or subtle question — not just validation
- Cannot mention SELLL or sales — this is relationship building, not pitching
- Should sound like Aaron: direct, knowledgeable, peer-to-peer
- Never sycophantic

**Example (good):**
> "The 90-day window is real — most new VPs underestimate how fast the inherited stack becomes their reputation. What's the one thing you wish you'd known before week one?"

**Example (bad):**
> "Great insight! Really resonates with what we've been seeing too."

---

## Approval Workflow

1. `linkedin-automation` skill generates comments and writes to today's file here
2. Aaron opens the file, reviews each comment, marks Approve or Skip
3. Aaron runs `linkedin-automation` → "post approved comments" to trigger Expandi
4. Expandi posts approved comments; skips the rest
5. File is archived after processing

---

## Archive

Processed comment files are moved to `comment-queue/archive/[YYYY-MM].md` (monthly rollup) after posting.
