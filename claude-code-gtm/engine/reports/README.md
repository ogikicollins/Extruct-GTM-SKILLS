# Reports — SELLL.io
> Generated by: `signal-monitor` skill | Frequency: every 72-hour scan cycle
> Purpose: Signal scan output, weekly pipeline health, and performance summaries.

---

## Report Types

### 1. Signal Scan Report
**Frequency:** Every 72 hours (automated)
**File pattern:** `signal-scan-[YYYY-MM-DD].md`
**Contents:**
- New signals detected (companies, trigger type, hypothesis match)
- Watch accounts checked and status
- Re-engagement triggers fired
- New accounts added to signal-queue.md
- Accounts that failed ICP check (with reason)

### 2. Weekly Pipeline Report
**Frequency:** Every Monday
**File pattern:** `weekly-[YYYY-WW].md`
**Contents:**
- Active campaigns: contact count, emails sent, opens, replies
- Reply breakdown by category (HOT/WARM/NOT_NOW/etc.)
- Meetings booked this week
- Re-engagement queue status
- Fatigue suppressed accounts added

### 3. Campaign Launch Report
**Frequency:** On every new Instantly campaign upload
**File pattern:** `launch-[campaign-name]-[YYYY-MM-DD].md`
**Contents:**
- Campaign name and hypothesis
- Contact count by tier and persona
- Sequence variants used
- Priority Personalization List size
- HeyGen videos queued vs. ready
- Pre-engagement schedule (T-3/T-2/T-1 dates per contact)

---

## Archive

Reports are kept for 90 days. Older reports are summarised into the monthly institutional memory log in `brain/institutional-memory/`.
