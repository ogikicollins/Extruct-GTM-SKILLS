# Daily Operating Runbook — SELLL.io Revenue Engine
> Brain Layer: Execution | Updated: 2026-06-21
> The engine generates opportunities. The human closes them. This is the 30-minute daily protocol that keeps every layer alive.

---

## Morning Session — 15 minutes (before 9 AM)

Run in this exact order. Do not skip to Step 2 until Step 1 is fully clear.

### Step 1: Reply Triage (5 min)

Open inbox. Every reply received since yesterday EOD gets classified before anything else.

```
□ Open reply-routing.md
□ Classify each reply by category (1–12)
□ HOT (Category 1): respond NOW — the 2-hour window started when they replied, not when you woke up
□ WARM / CONDITIONAL / OBJECTING (Cat 2, 3, 4): queue for response within 4 hours
□ All other categories: queue for same-day or 24h response per speed rules table
```

**Rule:** Never move to Step 2 until every HOT reply has been sent.

---

### Step 2: Signal Check (3 min)

```
□ Open LinkedIn notifications — who engaged with last post? Any ICP-fit profile views?
□ Check Google Alerts (or signal-monitor output) for target accounts: funding, exec hires, press
□ Any new trigger events? → route to trigger-playbooks.md immediately
□ Log new signals in state.md Signal Events table before closing this step
```

If a compound signal is detected (2+ hypotheses firing at same company) → escalate to CRITICAL on priority-board.md before continuing.

---

### Step 3: CRM Pulse (3 min)

```
□ Open HubSpot + crm-hygiene.md daily pulse checklist
□ Run all 6 items on the daily pulse
□ Any deals that moved stages overnight? → update state.md Layer 5 table
□ Any meetings booked? → add to pre-call brief queue (call-intelligence.md)
□ Any deals stalled > 5 days with no activity? → flag as today's priority action
```

---

### Step 4: Today's Execution Queue (4 min)

Pull today's action list from:

```
□ priority-board.md — any Tier 1 accounts to action today?
□ call-queue.md (if exists) — any Day 3 or Day 8 calls on the schedule?
□ video-log.md — any Looms queued for today's send?
□ re-engagement-queue.md — any contacts scheduled for today's follow-up?
□ Active sequence schedule — which emails send today? Are the personalization fields filled?
```

**Output:** A written list of today's 5 most important revenue actions. If it's not on the list, it doesn't happen today.

## Emergency Protocols

### Wrong person received an email / personalization error
1. Stop the sequence immediately in Instantly
2. Send an apology within 30 minutes: "I made an error in my last email — [what was wrong]. Apologies."
3. Log the error in state.md
4. Fix the template before relaunching

### Deliverability emergency (bounce > 2% or spam flag)
→ deliverability-rules.md → Emergency Protocol section
→ Pause ALL sending immediately. Diagnose before resuming.

### HOT lead replies while you're unavailable
Set an auto-responder: "On a call — you can book directly here: https://cal.com/collins-ogiki-x4fokk/30min. Back to you within the hour."
The 2-hour rule depends on you. Do not let the brain down.

### Meeting no-show
→ meeting-automation SKILL.md → No-Show Protocol
→ 15-minute buffer email → same-day reschedule → 48-hour final

### CRM goes down
Log manually in Notes. Import to HubSpot same day. Never let a deal exist only in memory overnight.

## The Sequence of Tasks Rule

**Run the runbook before social media. Before Slack. Before email newsletters.**

The engine works on a signal-triggered model — signals get cold the longer they sit. The morning protocol is the first revenue action of the day, not the fifth.
